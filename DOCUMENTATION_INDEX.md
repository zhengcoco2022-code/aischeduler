## How to Keep Documentation Updated

### When Adding Features
1. Update the relevant feature section
2. Add API endpoint (if applicable)
3. Update troubleshooting if new issues emerge
4. Update examples with new use cases

### When Changing APIs
1. Update [API_REFERENCE.md](./API_REFERENCE.md)
2. Add migration guide if breaking change
3. Update code examples
4. Update relevant guide sections

### When Fixing Issues
1. Add solution to troubleshooting section
2. Update relevant guide section
3. Add code comments if complex

### When Deploying
1. Update deployment checklist
2. Add new environment variables
3. Document new requirements

## Getting Help

### Troubleshooting
1. Check [GUIDE.md](./GUIDE.md#troubleshooting) troubleshooting section
2. Check [DEPLOYMENT.md](./DEPLOYMENT.md#troubleshooting) deployment troubleshooting
3. Review browser console (F12)
4. Check server logs

### Questions
1. Check relevant documentation file
2. Review code comments
3. Check API examples
4. Review component source code

### Bugs
1. Describe the issue clearly
2. Include steps to reproduce
3. Share error messages
4. Mention browser/OS version
5. Attach relevant screenshots

1. Check if already implemented
